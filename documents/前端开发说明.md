# 前端开发说明

这份说明只面向 `微信小程序前端` 开发同学，目标是最快把本地开发环境跑起来。

## 1. 你需要安装什么

前端最少只需要这 3 样：

1. `Git`
2. `微信开发者工具`
3. `JDK 17`

说明：

- `Git` 用来拉代码、提交代码
- `微信开发者工具` 用来运行和调试 `miniprogram-1/`
- `JDK 17` 不是为了写前端代码，而是为了你本地联调后端 Demo

当前仓库里：

- 前端目录是 `miniprogram-1/`
- 后端目录是 `demo/`
- 小程序请求地址已经配置在 `miniprogram-1/utils/config.js`
- 当前默认后端地址是 `http://127.0.0.1:18080`

## 2. 不需要安装什么

当前前端不是 React/Vue Web 项目，所以你现在不需要安装这些：

- `Node.js`
- `npm`
- `pnpm`
- `yarn`

如果后续团队决定把管理端或官网拆成 Web 项目，再单独补这部分环境。

## 3. 当前机器检查结果

我已经在这台机器上确认：

- `java -version` 正常，当前是 JDK 17
- `git --version` 正常
- 系统里有 `WeChat.app`

还没有全局 `mvn`，但这不是问题，因为后端目录自带 `demo/mvnw`

## 4. 前端项目怎么打开

1. 打开微信开发者工具
2. 导入目录时选择 `miniprogram-1/`
3. 不要选择项目根目录 `2026SE/`
4. 如果你有自己的小程序 `AppID`，填入 `miniprogram-1/project.config.json`
5. 如果你暂时没有真实 `AppID`，导入项目时在微信开发者工具里选择测试号

当前我已经为本地补好了：

- `miniprogram-1/project.config.json`

你只需要把里面的：

- `YOUR_APP_ID_HERE`

替换成你自己的真实 `AppID` 即可。

## 5. 联调时怎么启动后端

仓库当前默认端口不是 `8080`，而是：

- `18080`

默认 profile 是：

- `h2`

也就是说，最省事的本地联调方式是直接用内存数据库跑 Demo。

在项目根目录执行：

```bash
cd demo
sh mvnw spring-boot:run
```

启动成功后，日志里应该能看到：

- `The following 1 profile is active: "h2"`
- `Tomcat started on port 18080`

## 6. 小程序联调前要确认什么

### 后端地址

文件：

- `miniprogram-1/utils/config.js`

当前配置：

```js
const BASE_URL = 'http://127.0.0.1:18080'
```

如果你改了后端端口，这里也要一起改。

### 开发者工具设置

在微信开发者工具里建议确认：

1. 打开“详情”
2. 进入“本地设置”
3. 勾选“不校验合法域名、web-view、TLS 版本以及 HTTPS 证书”

## 7. 你现在可以直接做的事

1. 用微信开发者工具导入 `miniprogram-1/`
2. 把 `project.config.json` 里的 `appid` 改成你的
3. 在终端进入 `demo/` 目录
4. 运行 `sh mvnw spring-boot:run`
5. 回到微信开发者工具点击“编译”

## 8. 常见坑

### 打不开小程序项目

优先检查：

- 你导入的是不是 `miniprogram-1/`
- `project.config.json` 里的 `appid` 是否已处理

### 页面空白或接口报错

优先检查：

- 后端是否已经启动
- 后端是否跑在 `18080`
- `BASE_URL` 是否还是 `http://127.0.0.1:18080`
- 开发者工具是否关闭了域名校验

### 401 未登录

当前 Demo 登录走的是开发简化流程，前端会用：

- `code=demo`

去调用后端登录接口。
